## 1. Community Vision & Values

### Vision

Justice OS aims to be the world's most impactful open-source community in legal technology — a global movement of developers, lawyers, advocates, and community organizers united by the belief that access to justice is a human right.

By 2027, we envision:
- 500+ active contributors across 30+ countries
- 72 actively maintained open-source modules
- A recognized brand in legal tech, open source, and social impact
- 10,000+ people using Justice OS-powered tools to navigate legal challenges

### Core Values

| Value | Description |
|-------|-------------|
| **Justice** | Every contribution serves people who need legal help |
| **Openness** | Code, decisions, and roadmaps are transparent |
| **Inclusion** | Barriers to contribution are actively removed |
| **Excellence** | We hold each other to high standards with empathy |
| **Sustainability** | We build for the long-term, not quick wins |

### Community Manifesto

> "We believe legal technology should serve everyone, not just those who can afford it.
> We write code in service of justice. We review PRs in service of people.
> Every issue closed is a family helped. Every module shipped is a barrier removed."

## 2. Contributor Tiers

### Tier Structure

```
Founder (6+)          ← Shaping the project direction
    ↑
Maintainer (15+)      ← Leading modules and teams
    ↑
Contributor (3+)      ← Steady, meaningful contributions
    ↑
Newcomer (0)          ← First issues, onboarding, learning
```

### Tier Details

**Newcomer**
- Who: First-time contributors; students; exploring the project
- Requirements: None — just show up!
- Rights: Can comment, open issues, submit PRs
- Recognition: Welcome message; newcomer badge on profile
- Support: Assigned an onboarding buddy within 48 hours

**Contributor**
- Who: Has merged 3+ PRs or equivalent contributions
- Requirements: Sustained engagement over 30+ days
- Rights: Reviewer on PRs in their area; invited to monthly calls
- Recognition: Contributor badge; listed in CONTRIBUTORS.md
- Perks: Swag pack; invitation to private contributor Slack channel

**Maintainer**
- Who: Owns or co-owns one or more modules
- Requirements: 15+ merged PRs or equivalent; nominated by existing maintainer
- Rights: Merge PRs in their module; vote on module roadmap; attend leadership calls
- Recognition: Listed as module maintainer in README; profile on website
- Perks: Annual stipend (when funded); conference speaking support; annual summit travel

**Founder**
- Who: Core team + long-term contributors with outsized impact
- Requirements: Invitation only; sustained contribution for 12+ months
- Rights: Vote on project-wide decisions; access to all private channels; advisory role
- Recognition: Founder badge; listed on About page; named in press releases
- Perks: Equity participation (if applicable); advisory compensation; strategic input

## 3. Contribution Pathways

Justice OS explicitly values non-code contributions equally with code contributions.

**Code Contributions:**
- Bug fixes (`good first issue` label)
- Feature development (per roadmap)
- Performance improvements

- Test coverage improvements
- Module creation (72-project ecosystem)

## 4. Mentorship Program Design

### Goals
- Reduce time-to-first-PR from 2 weeks to 3 days
- Retain 60% of newcomers past their first contribution
- Build pipeline of future maintainers

### Structure

**Onboarding Buddy Program**
- Every new contributor is assigned a buddy within 48 hours
- Buddy commits to 1-hour check-in within the first week
- Buddy answers questions for the first 30 days
- Buddy program is voluntary; buddies earn 20 points/mentee

**Module Mentorship**
- Each module has a designated mentor (typically the maintainer)
- Newcomers interested in a module are connected with the mentor
- Monthly 30-minute pairing sessions available

**Leadership Mentorship**
- Contributors aspiring to become maintainers are paired with a current maintainer
- 3-month structured program: shadowing → ownership → independence
- Includes: PR review coaching, community management skills, roadmap planning

### Mentor Expectations
- Respond to mentee questions within 24 hours (business days)
- Monthly 30-minute video call
- Complete monthly feedback form (5 minutes)

## 5. Community Events Strategy

### Monthly Events

**Monthly Community Call (1st Thursday of month, 60 min)**
- Format: Video call (Zoom/Meet), recorded and posted
- Agenda:
  - 5 min: Welcome + newcomer introductions
  - 15 min: Month in review (metrics, PRs, news)
  - 20 min: Featured contributor spotlight
  - 15 min: Module showcase or demo
  - 5 min: Q&A and announcements

**Monthly Hack Session (3rd Saturday, 3 hours)**
- Focus: Close `good first issue` backlog
- Format: Open video session; maintainers available for questions
- Goal: Each session closes 10+ issues and merges 5+ PRs

### Quarterly Events

**Quarterly Planning Summit (Virtual, 3 hours)**
- Attendance: Maintainers + Contributors
- Agenda:
  - Review last quarter metrics
  - Approve next quarter roadmap
  - Recognize top contributors
  - Plan community initiatives

**Quarterly Newcomer Bootcamp (4 sessions × 90 min)**
- Audience: Developers new to Justice OS
- Content: Architecture, development setup, first contribution walkthrough
- Instructors: Maintainers + Founders

### Annual Events

**Annual Justice OS Summit**
- Format: 2-day virtual conference + optional in-person hub
- Attendance target: 500+ Year 1, 2,000+ Year 3
- Program:
  - Keynotes from founders + community leaders
  - Technical workshops
  - Module showcases
  - Community awards ceremony
  - Partner announcements

## 7. Code of Conduct & Governance

### Code of Conduct Summary

Justice OS is committed to providing a welcoming and inclusive environment. We do not tolerate:
- Harassment or discrimination of any kind
- Dismissive or hostile communication
- Deliberate exclusion or gatekeeping
- Sharing private information without consent

Full Code of Conduct: [LINK TO CODE_OF_CONDUCT.md]

**Enforcement:**
- Minor violations: Private warning from a maintainer
- Repeated violations: Temporary ban (30–90 days)
- Severe violations: Permanent ban, reported to GitHub

**Reporting:** report@justiceos.dev (reviewed by founding team within 24 hours)

### Governance Model

**Decision Types:**

| Decision | Who Decides | Process |
|----------|-------------|---------|
| Bug fix / routine PR | Module maintainer | Lazy consensus (1 approval + 72h) |
| New feature (in-module) | Module maintainer | Discussion + vote (maintainers) |
| New module | Founding team | RFC process + community vote |
| Project-wide policy | Founding team + Founders | RFC + Steering Committee vote |
| Financial decisions | Founding team | Founder vote |

**RFC Process (Request for Comment):**
1. Author opens GitHub Discussion with `RFC:` prefix
2. 14-day comment period
3. Maintainer synthesizes feedback
4. Decision published with rationale

## 9. Measurement Metrics

### Community Health Dashboard

**Weekly Metrics:**
- New contributors (first PR or first comment)
- PRs opened / merged / closed without merge
- Issues opened / closed
- Discord active users

**Monthly Metrics:**
- Contributor retention rate (% of last-month contributors who contributed again)
- Time-to-first-PR for newcomers (target: < 3 days)
- PR review turnaround (target: < 48 hours)
- Community call attendance
- Newsletter open rate (target: 35%+)

**Quarterly Metrics:**
- Contributor tier distribution (newcomer / contributor / maintainer / founder)
- Module health score (active maintainer, recent commits, CI passing)
- Community NPS (survey)
- Geographic diversity of contributors

### Year-1 Targets

| Metric | Target |
|--------|--------|
| Active contributors (monthly) | 50+ |
| Total contributors (cumulative) | 200+ |
| Countries represented | 20+ |
| Module maintainers | 30+ |
| Community call attendance | 50+ avg |
| Newsletter subscribers | 2,000+ |
| Discord members | 1,000+ |
| PR review time (avg) | < 48 hours |
| Newcomer → Contributor conversion | 30% |

### Tools for Measurement
- Orbit.love or Common Room: contributor relationship management
- GitHub Insights: PR / commit / issue metrics
- Discord analytics: active users, message volume
- Mailchimp / Ghost: newsletter analytics
- Annual Community Survey: NPS, satisfaction, barriers
